Create or update a command object from a TOKEN_CMD token.

If current_cmd is NULL a new t_command is allocated and initialized with the token content as the command name. Otherwise the function will add the token content as an argument to the existing command.

Parameters

Finalize the current command on a pipe token.

Pushes the provided current_cmd into the commands list. This is called when a pipe token is encountered to close the current command and prepare for the next one.

Handle a redirection token and attach the redirection to the current command object.

Reads the next token (if present) as the redirection target. If the current command has no name yet, consumes an additional token to use as the command name. Supported redirect tokens: <, >, >>, << (heredoc).

Parameters
tokToken representing the redirection operator.
file_tokOptional token representing the redirection target (may be NULL).
current_cmdCommand object to attach the redirection to; if NULL a new command will be created.
itIterator used to consume extra tokens when needed.
Returns
Pointer to the updated or newly created t_command, or NULL on error.
